# طلبات لـ Claude — لتطوير أدوات ومراجع لمشروع SAJCO

---

## الطلب 1: Agent للبحث السريع في الكود

يا Claude، أبغى تسوي لي Agent صغير أو Skill وظيفته إنه يكون خبير في قاعدة كود مشروع SAJCO. الـ Agent هذا لازم يكون عنده القدرة إنه يبحث بسرعة عن أي دالة، متغير أو نمط في المشروع، ويعطيني الإجابة بشكل منظم. مثلاً لو سألته "وين موجودة دالة addNotification؟" أو "أيش دوال render المرتبطة بصفحة maintenance؟" أو "أيش كل حالات الـ status للـ custodies؟"، يرد علي بسرعة بدون ما يحتاج أستخدم grep وحدي. يفضل تكون لغة العمل عربي.

## الطلب 2: Auto Sync Checker بين العربي والإنجليزي

يا Claude، سوّي لي Script صغير (PowerShell أو Node) وظيفته يفحص مجلد المشروع ويتأكد إن الملفات العربية والإنگليزية متطابقة في الدوال. يعني يقارن كل زوج files زي maintenance.html مع en/maintenance.html ويدور على دوال أو أقساط ناقصة أو زايدة. إذا لاقى فرق، يطبع تحذير. هذا يخليني ما أنسى أطبق أي إصلاح على النسختين.

## الطلب 3: State Machine Reference

يا Claude، أبغى منك مستند اسمه STATES.md يشرح بالضبط حالات كل كيان في المشروع (أجهزة، عهد، طلبات، مستخدمين) وانتقالاتها. مثلاً للجهاز: الحالة warehouse تتحول لـ assigned بعدين ممكن maintenance أو needs_calibration، وكل انتقال أي دالة تسويه وأي إشعار ينرسل وأي تحديث لـ Supabase. يفضل بالعربي ويكون واضح كمرجع سريع.

---

## الطلب 4: Debug Entry Points في AGENTS.md

يا Claude، أبغيك تضيف قسم في AGENTS.md اسمه "Debug Entry Points" ويحتوي على جدول لكل مشكلة معروفة في المشروع. كل مشكلة يكتب لها أول 3 خطوات أتفقدها بالترتيب عشان أشخصها بسرعة. مثلاً:
- مشكلة: زر القبول/الرفض ما يظهر → أول شيء أتأكد من وجود السجل في db.custodies، ثاني شيء أتأكد من empId، ثالث شيء أتأكد من الـ dedup في _execLoadRemoteDB.
- مشكلة: بيانات تختفي بعد reload → أول شيء localStorage، ثاني شيء Supabase sync، ثالث شيء RLS error.
